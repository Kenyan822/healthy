# 機能要件・非機能要件定義書

## 1. プロジェクト概要

### 1.1 プロジェクト名
チェーン店ヘルシー検索サイト

### 1.2 概要
トレーニーやダイエット中の女性をターゲットに、チェーン店の食事のPFC（タンパク質・脂質・炭水化物）やヘルシー度合いを数値化してまとめるWebサイト。プログラマティックSEOを活用し、数千〜数万ページを自動生成してロングテールキーワードでの検索流入を独占する。

### 1.3 目的
- データドリブンなSEOサイトの構築
- チェーン店の栄養成分データを構造化
- PFCバランス・ヘルシー度合いでの検索・絞り込み機能の提供
- 検索流入から広告収入・アフィリエイト収入を得る

### 1.4 ターゲットユーザー
- トレーニー（筋トレをしている人）
- ダイエット中の女性
- 健康志向の人
- 外食が多いビジネスパーソン

## 2. 機能要件

### 2.1 キーワードリサーチ機能

#### 機能概要
Google Keyword Plannerを活用し、検索ボリューム・競合分析を行い、ロングテールキーワードを抽出する。

#### 詳細要件

| 要件ID | 要件名 | 説明 |
|--------|--------|------|
| KW-001 | 検索ボリューム取得 | Google Keyword Planner APIで月間検索ボリュームを取得 |
| KW-002 | 競合分析 | 競合サイト数、ドメイン権威性を分析 |
| KW-003 | ロングテール抽出 | チェーン店名×目的の組み合わせでキーワード生成 |
| KW-004 | 検索意図分類 | 情報収集、比較、購入などに分類 |

#### 狙うキーワード戦略

**A. チェーン店名 × 目的（最優先）**
- 「大戸屋 高タンパク メニュー」
- 「マクドナルド 低脂質」
- 「すき家 ダイエット 組み合わせ」
- 「スタバ 低糖質 フード」

**B. 駅名 × チェーン・条件**
- 「渋谷駅 高タンパク ランチ」
- 「新宿 ヘルシー チェーン店」
- 「東横線 一人飯 ダイエット」

**C. メニュー単体（ロングテール）**
- 「チキンかあさん煮定食 PFC」
- 「牛丼ライト カロリー」
- 「ビッグマック タンパク質」

**D. 栄養素 × 目標量**
- 「タンパク質 30g メニュー 組み合わせ」
- 「糖質 50g 以下 チェーン店」
- 「脂質 20g 以下 メニュー」

### 2.2 データソース管理機能

#### 機能概要
チェーン店の栄養成分データを収集・管理し、独自スコアを計算する。

#### データソース
- チェーン店公式データ（栄養成分表PDF、Webページ）
- スクレイピング（規約遵守の範囲内）
- 手動入力（初期は主要5-10社）

#### 詳細要件

| 要件ID | 要件名 | 説明 |
|--------|--------|------|
| DS-001 | メニューデータ取得 | 公式サイトから栄養成分を取得 |
| DS-002 | スコア計算 | マッスルスコア、ケトスコア、ヘルシースコアを計算 |
| DS-003 | データ更新 | 月1回の定期更新 |
| DS-004 | 季節限定対応 | 季節限定メニューのフラグ管理 |

#### スコア計算ロジック

```
マッスルスコア = タンパク質(g) ÷ カロリー(kcal)
  → 高タンパク・低カロリーほど高スコア

ケトスコア = 100 - (炭水化物 × 2)
  → 糖質が低いほど高スコア（50g以下で有効）

ヘルシースコア = (タンパク質 × 2) - (脂質 × 1.5) - (炭水化物 × 0.5)
  → 総合的な健康指標
```

### 2.3 コンテンツ生成機能

#### 機能概要
テンプレートベースとAI生成のハイブリッドでページコンテンツを生成する。

#### 生成方法
- テンプレートベース: データをテンプレートに埋め込む
- AI生成: ChatGPT API / Claude APIでコンテンツ生成
- ハイブリッド: テンプレート + AIで品質向上

#### 詳細要件

| 要件ID | 要件名 | 説明 |
|--------|--------|------|
| CG-001 | テンプレート生成 | 定型フォーマットでのページ生成 |
| CG-002 | AI生成 | 自然な文章でコンテンツ生成 |
| CG-003 | メタデータ生成 | title, description, keywordsの自動生成 |
| CG-004 | YMYL配慮 | 断言を避け、事実の提示に徹する |

#### ページタイプ
1. チェーン店×目的ページ（例: `/大戸屋/高タンパク`）
2. 駅×条件ページ（例: `/渋谷駅/高タンパク-ランチ`）
3. メニュー詳細ページ（例: `/メニュー/チキンかあさん煮定食`）
4. 組み合わせ提案ページ（例: `/組み合わせ/タンパク質30g`）

### 2.4 メニュー組み合わせ提案機能

#### 機能概要
PFCの目標値を達成できるメニューの組み合わせを自動計算・提案する。

#### ユースケース
1. タンパク質優先: 「タンパク質30g摂取したい」
2. 糖質制限: 「糖質50g以下に抑えたい」
3. 脂質制限: 「脂質20g以下に抑えたい」
4. 複合条件: 「タンパク質30g以上、かつ糖質50g以下」

#### 詳細要件

| 要件ID | 要件名 | 説明 |
|--------|--------|------|
| MC-001 | 単一栄養素目標 | 指定栄養素の目標値達成 |
| MC-002 | 複合条件 | 複数の栄養素条件を同時に満たす |
| MC-003 | 価格・カロリー考慮 | 最大価格、最大カロリーの制約 |
| MC-004 | 組み合わせスコア | 最適な組み合わせの評価・ランキング |

## 4. リスク管理

### 4.1 Google品質ガイドライン

**リスク**: 自動生成コンテンツがスパム判定される

**対策**:
- 高品質なコンテンツ生成（AI活用）
- 実際の公式データを使用
- 定期的な更新
- 断言を避け、事実の提示に徹する

### 4.2 データ更新の負担

**リスク**: 季節限定メニューの頻繁な更新

**対策**:
- グランドメニュー（定番）に絞る
- 季節商品は余裕が出てから対応
- 月1回の更新運用

### 4.3 競合の参入

**リスク**: 成功後の競合増加

**対策**:
- 差別化（独自スコア、UI/UX、データ正確性）
- 継続的な改善
- 複数の収入源確保

### 4.4 インデックス遅延

**リスク**: 検索結果に表示されるまでの時間

**対策**:
- Google Search Consoleでの早期インデックス申請
- 長期的な視点での取り組み
- 複数サイトの並行運営
